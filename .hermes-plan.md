# PoroScope — Development Plan

> **Goal:** Turn PoroScope from a CLI porosity tool into a comprehensive, open-source **materials microstructure analysis suite** for academic researchers, with napari GUI, documentation, and PyPI publishing.

**Architecture:** Modular scientific Python toolkit. Core analysis modules (segmentation, measurements, preprocessing) + CLI (typer) + napari plugin (optional GUI) + docs (MkDocs). TDD throughout.

**Tech Stack:** Python 3.10+, scikit-image, numpy, pandas, napari (GUI), typer (CLI), pytest (tests), MkDocs (docs)

---

## Phase 1 — Foundation & v0.2 Release

### Task 1: Expand analysis capabilities

**Objective:** Add grain size analysis, phase fraction, and particle analysis modules alongside existing porosity pipeline.

**Files:**
- Create: `src/poroscope/grains.py` — grain size analysis (watershed segmentation)
- Create: `src/poroscope/phases.py` — phase fraction analysis (multi-class thresholding)
- Create: `src/poroscope/particles.py` — particle analysis (size distribution, shape factors)
- Create: `tests/test_grains.py`
- Create: `tests/test_phases.py`
- Create: `tests/test_particles.py`
- Modify: `src/poroscope/cli.py` — add new CLI commands
- Modify: `src/poroscope/pipeline.py` — add new pipeline functions

### Task 2: napari plugin

**Objective:** Build a napari plugin so researchers can use PoroScope interactively with their microscopy images.

**Files:**
- Create: `src/poroscope/napari_plugin.py` — napari dock widget
- Create: `src/poroscope/_tests/test_napari_plugin.py`
- Modify: `pyproject.toml` — add napari deps + entry points
- Create: `src/poroscope/napari.yaml` — napari plugin manifest

### Task 3: Documentation site

**Objective:** Professional MkDocs documentation with API reference, user guide, examples.

**Files:**
- Create: `docs/` directory with MkDocs structure
- Create: `docs/index.md`, `docs/guide/`, `docs/api/`, `docs/examples/`
- Create: `mkdocs.yml`
- Modify: `pyproject.toml` — add docs deps
- Create: `.github/workflows/docs.yml` — auto-deploy to GitHub Pages

### Task 4: PyPI publishing setup

**Objective:** Ready for `pip install poroscope`.

**Files:**
- Modify: `pyproject.toml` — finalize metadata, classifiers, long_description
- Create: `.github/workflows/publish.yml` — publish to PyPI on tag

## Phase 2 — Advanced Features

## Quick Commands

```bash
# Dev
cd /root/PoroScope && source .venv/bin/activate
pytest tests/ -v

# Test poroscope on SEM image
poroscope analyze examples/data/sem_samples/sem_znse_porous.jpg --pixel-size 0.5 --unit um --pores dark --output sem_results/

# Batch
poroscope batch examples/data/sem_samples/ --pixel-size 0.5 --unit um --output batch_results/
```
